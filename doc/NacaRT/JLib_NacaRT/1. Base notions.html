1.1 Nomemclature

Name Description
NacaRT Stands for Naca Runtime - Base library exposing a high level abstraction of methods supproting a cobol - like syntax. It's the runtime framework of Nacar
JLib Stands for Java Libraray - Base library exposing generic services internally used by NacaRT
Idea Internal nacaRT part, made of several packages, offering user interface services
NacaLib Internal NacaRT part, made of several packages, offering runtime support for cobol-like application programs
Cobol-like Name of the syntax of Cobol programs automatically transcoded in Java source files. This syntax has been design to be as close as possible to the original Cobol syntax. In particular, object orientation is out of scope from this syntax. The purpose is enable experimented Cobol developpers to maintain and develop the transcoded applications with a minimum amount of Java knowledge.
Program Java Executable binary code, represented in a .class
Program instance Instance of a program. Many instqances can be executed simulataeously
Variable (var) Defines a memory place where to store a value. A variable is owned by a single program instance, so that it's value is not not shared among all program instances.
Variable definition (vardef) Java objects defining the various characteristics of a given variable (name, hierarchical position, ...) It's shared among all instances of a given program. A variable owns a reference to it's variable definition.
Variable definition catalog Collection that groups all variabke definitions of a given program
Buffer Memory storage area where values of variables of a program instance are stored. A buffer is by default not shared by multiple program instances. There may ba excpetion to that rule.
Parent Cobol variables belongs to a hierarchy, which is defined by the variable definition depth level number. The parent of a given variable definition is the variable definition of a lowest depth level that precedes it.
Children A given variable definition may own a list of sub-variables. They are the children of the current variable definition.
Dimension Number of indexes that are needed to resolve a array access. An array of a dimension of 2 needs to indexes to find an item. The maximum dimension supported is 3.
Occurs owner Parent variable of the variable that owns an occurs definition.
Cobol exemple:
01 A OCCURS 3.
   05 B PIC X(10).
A is the occurs owner of B. That means that A defines the number of times that B is instanciated. As occurs can be imbricated, a variable may have multiple occurs owner.
Nibble Half byte = 4 bits. A byte occupied then 2 nibbles
Unicode char Alphanumeric supporting variables are stored internally as unicode String. Each char reserves 16 bits for it's internal representation. As of 01/04/07, only 8 bits ascii subset is used.
Edit Screen entry field. It has the same attributes as a CICS BMS map  entry field, but is used in an object oriented manner, instead of byte per byte
Form
Collection of Edits, representing a CICS BMS Map
MapRedefine
Objects that redefines a form. It enables occess to the Form field in an indexed manner
EdiInMapRedefine
Redefine of an Edit
